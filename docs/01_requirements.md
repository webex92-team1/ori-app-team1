# 🍳 FoodMatch（仮） 要件定義書

---

## 1. アプリ概要

- **目的（Why）**

  - 毎日の献立を考える負担を軽減する
  - 家にある食材を活用して、無駄を減らす
  - 料理初心者でも簡単にレシピを探せるようにする

- **ターゲット（Who）**

  - 一人暮らしの大学生
  - 料理初心者〜基本料理ができるレベルの人

- **利用シーン（When / Where）**

  - 学校・バイトから帰宅して「今日何食べよう？」となった時
  - 冷蔵庫を開けたとき
  - 外出前に「ある食材で作れるもの」を素早く調べたい時

- **提供価値（What）**

  - 食材を入力するだけで、レシピを自動で提案
  - 足りない材料も分かりやすく表示
  - お気に入りや履歴で料理の選択がラクになる

- **技術スタック（How）**

  - フロントエンド：React, JavaScript, Next.js
  - バックエンド：Firebase（Auth / Firestore）
  - 外部 API：Rakuten レシピ API
  - UI：Tailwind CSS, shadcn/ui
  - ホスティング：Firebase Hosting or Vercel

## 2. 機能要件（MVP）

| 優先度 | 機能名               | 概要                                            | ユーザーストーリー                                                   |
| :----: | :------------------- | :---------------------------------------------- | :------------------------------------------------------------------- |
|  ★★★   | 食材入力機能         | テキストで「卵 牛乳 玉ねぎ」などを入力          | 今日ある食材を入力し、レシピ検索に進む                               |
|  ★★★   | レシピ検索・提案機能 | Rakuten レシピ API から候補を取得し一覧表示     | 入力した食材で作れるレシピが並び、足りない食材も灰色表示されて分かる |
|  ★★★   | レシピ詳細ページ     | 材料・手順を確認できる                          | 作りたいレシピを確認しながら料理できる                               |
|  ★★☆   | ユーザー認証         | Google ログイン or メール・パスワードでログイン | 履歴とお気に入りが同期される                                         |
|  ★★☆   | お気に入り機能       | レシピを保存できる                              | よく作るレシピをすぐ見返せる                                         |
|  ★★☆   | 作成履歴機能         | 「作った」ボタンでレシピ名＋日付を保存          | 過去に作った料理を振り返られる                                       |
|  ★☆☆   | マイページ           | お気に入り・履歴を一覧表示                      | 最低限の UI で OK・後から強化可能                                    |

## 4. 想定ユーザーフロー

1. ユーザーがトップページにアクセス
2. 新規登録（メール・パスワード）or Google ログイン or 既存アカウントでログイン
3. 食材入力ページで「卵 玉ねぎ 牛乳」などを入力
4. レシピ一覧ページで候補を見る
5. 気になるレシピをクリック
6. レシピ詳細を確認して料理開始
7. 作ったら「作った！」ボタンで履歴保存
8. マイページでお気に入り・履歴を確認

## 5. 開発ルール（初心者向け）

- **ブランチ運用**

  - main：完成版
  - feature/〇〇：各自の作業用

- **コミットルール**

  - 1 機能＝ 1 コミット
  - 例：`feat: add recipe list UI`
    `fix: recipe API fetch error`

- **進捗共有**

  - 毎週 1 回、簡単な進捗共有（Discord/LINE/Notion）
  - GitHub Projects に ToDo を管理（任意）
